🩺 Diagnostics

Log Level

Minimum Log Level
Controls verbosity across all enabled categories. Information (normal events), Debug (detailed troubleshooting), Verbose (full internal tracing — may impact performance).

Logging Categories

Toggle which subsystems produce log output. Changes take effect immediately. Defaults shown below.

LLM API Calls
Full request payload (system prompt, messages, model, temp, max tokens — API keys REDACTED), response content, finish reason, token counts, latency (ms), streaming chunk count, HTTP error bodies and status codes.
Tier 1 Hotkey Pipeline
Hotkey detected → TextAction loaded → capture scope → UIA pattern → content captured (first 200 chars) → AI call dispatched → apply mode outcome (HWND injection vs clipboard fallback) → success/failure with reason.
Tier 2 Command Bar
Alt+Space triggered → query text → AI call dispatched → pop-out/inline state transitions → elevation to Studio → dismissal (saved as transient thread).
Database
Slow queries (>100ms threshold: SQL + duration), EF Core migration execution (name + ms), VACUUM timing and space reclaimed, FTS5 search terms and result counts.
Wiki & File System
File watcher events (create/modify/delete with path), indexing runs (files scanned, duration, errors), git auto-commit (files changed, commit hash), backup operations (file count, total size, duration).
WebSocket
Client connections/disconnections, message counts (sent/received), auth attempts (success/failure — token redacted).
Startup & Shutdown
DI container build time, migration application (name + ms), key service initialization order and per-service timing, graceful shutdown sequence (services stopped in order).
System Integration
Global hotkey registration (success/failure, conflicts), system tray interactions (left-click, menu selection), clipboard capture/restoration, DPI change events, screenshot operations.
