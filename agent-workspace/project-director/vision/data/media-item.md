# MediaItem — Data Entity

## Description
A MediaItem is a binary media file (image, audio, video) that was either uploaded by the user, captured via webcam, or generated by AI. All media is stored on disk in the app's data directory and tracked in the database.

## Attributes

| Attribute | Type | Required | Constraints | Description |
|-----------|------|----------|-------------|-------------|
| id | string (UUID) | Yes | Unique | Primary identifier |
| filename | string | Yes | Original or generated filename | File name |
| filePath | string | Yes | Absolute path on disk | Where the file is stored |
| mediaType | enum | Yes | Image, Audio, Video | Type of media |
| mimeType | string | Yes | e.g., "image/png", "audio/mp3" | MIME type |
| fileSize | integer | Yes | In bytes | File size |
| source | enum | Yes | UserUpload, AIGenerated, WebcamCapture, Screenshot | Where media came from |
| threadId | string (FK) | Yes | References ChatThread | Chat where media was created |
| messageId | string (FK) | No | References Message | Message containing the media |
| generatedPrompt | string | No | Only for AI-generated media | Prompt used to generate |
| createdAt | datetime | Yes | Auto-set | Creation timestamp |
| isSavedToDisk | boolean | Yes | Default: false | Whether user explicitly saved to external location |
| isSavedToWiki | boolean | Yes | Default: false | Whether added to wiki via N5 |

## Lifecycle

### Create
- **User upload:** Drag-drop files (C9) or camera capture (C22). Copied to app data directory.
- **AI generation:** Image generation (G4) or audio generation (G5). Saved to app data directory.
- **Screenshot:** Future T4 feature.

### Update
- Media content is immutable. Only metadata updated (e.g., isSavedToDisk).
- Delete from library (G3): soft-delete? Or just remove from library view but keep for chat? (Architect decision)

### Delete
- Cascading: when parent ChatThread deleted AND isSavedToDisk=false AND isSavedToWiki=false → file deleted from disk (O5).
- Manual: user deletes from Media Library (G3).

## Relationships
- **belongs to** ChatThread (via threadId)
- **optionally belongs to** Message (via messageId)
- **may be referenced by** Wiki File (if saved to wiki)

## UI Visibility
- Chat conversation (G6) — Rendered inline in messages
- Media Library (G1) — Grid thumbnails with filtering
- Media Actions (G3) — View, download, copy, delete
