# Usage & Pricing Dashboard — Feature Spec

## What the User Accomplishes
The user views comprehensive usage statistics — token counts and estimated costs across all providers, models, and chats. Filterable by time range, provider, and chat. Charts visualize usage trends. Budget alerts prevent overspending.

## Trigger
- Studio sidebar → "Usage" nav item
- Settings → "Usage & Pricing" section

## Detailed Behavior

### S1. Usage Overview Screen
- **Access:** Dedicated screen accessible from Studio sidebar
- **Summary Cards (top):** Total tokens this month, total cost this month, most used model, most used Persona
- **Layout:** Cards at top, charts in middle, per-chat table at bottom

### S2. Time Range Filters
- **Presets:** Today, This Week, This Month, Custom Date Range, All Time
- **Default:** This Month
- **Custom Range:** Date picker with start and end dates
- All charts and tables update to reflect selected range

### S3. Usage Charts
- **Line Chart:** Token usage over time (X-axis: dates, Y-axis: token count). Separate lines for input tokens and output tokens.
- **Bar Chart:** Cost over time (X-axis: dates, Y-axis: estimated cost in USD).
- **Pie Chart — By Provider:** Token distribution across providers.
- **Pie Chart — By Model:** Token distribution across models.

### S4. Per-Chat Breakdown
- **Table Columns:** Chat Title, Date, Model(s) Used, Token Count (Input + Output), Estimated Cost, Persona
- **Sort:** Click column headers to sort
- **Click Chat:** Opens that chat in Studio
- **Empty State:** "No chat activity in the selected time range."

### S5. Budget Alerts
- **Monthly Budget:** User sets a monthly spending limit in Settings → Pricing (in USD)
- **Warning Threshold:** Configurable percentage (default: 80%). When reached: non-intrusive warning in Studio: "You've used 80% of your monthly budget ($[spent] of $[limit])."
- **Block Threshold:** Optional. When 100% reached: block further API calls. "Monthly budget reached. API calls are blocked until [1st of next month]. Adjust budget in Settings."
- **Reset:** Budget resets on 1st of each month
- **Disable:** Budget alerts can be disabled entirely

## Data
- [`data/usage-record.md`](data/usage-record.md) — token counts, costs per message

## Success/Failure States
- **No Data:** "No usage data yet. Start chatting to see your usage statistics."
- **Pricing Not Configured:** Cost charts show "Cost data unavailable — configure pricing in Model Configurations (B2)."
- **Budget Warning:** Non-intrusive banner: "Budget alert: [X]% of monthly limit used."

## Permissions
- Single-user app.

## Interactions
- S1-S4 read from usage records generated by C11 (token counting)
- S5 references B2 (model pricing configuration)
- S4 navigates to chats via L1 (sidebar)
